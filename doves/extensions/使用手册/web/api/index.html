Server API 路由速查

白鸽 Server 的完整路由参考。所有路由通过 Noise NX 加密 TCP（端口 3003）访问，无明文 HTTP。

协议说明：虽然路由格式类似 REST API，但传输层是 Noise NX 加密 TCP，不是 HTTP。内部 Express 仅绑定 localhost。

Public（无需认证）

Method Path 说明
GET /ping 连通性检查，返回 pong
GET /health 健康检查（含依赖状态）
GET /ready 就绪检查（是否可接受请求）
GET /api/versions API 版本信息
GET /api/protocol-doc 协议文档
POST /auth/register 用户注册
POST /auth/login 用户登录，返回 JWT
POST /auth/admin 管理员登录
POST /auth/anonymous 匿名登录（受限功能）
POST /api/im/:platform/webhook IM 平台 Webhook 接入（微信/钉钉/飞书）

Dove Authenticated（API Key 认证）

Method Path 说明
POST /api/dove/claim-task 原子任务认领（findOneAndUpdate）
POST /api/dove/task/heartbeat 任务心跳（证明鸽子活跃）
POST /api/dove/task/result 提交任务执行结果
POST /api/dove/task/release 释放任务（放弃执行，回归队列）
POST /api/dove/register 鸽子注册/上线
GET /api/dove/config 获取鸽子配置
POST /api/dove/capabilities 上报能力声明
* /api/dove/mcp/* MCP 协议相关操作
* /api/dove/extensions/* 扩展管理（安装/更新/查询）

认领是原子操作：/api/dove/claim-task 使用 MongoDB findOneAndUpdate 保证同一任务不会被重复认领。

Admin Only（管理员 + IP 白名单）

Method Path 说明
* /api/admin/users 用户管理（列表/禁用/删除）
* /api/admin/doves 鸽子管理（状态/配置/禁用）
* /api/admin/tasks 全局任务管理（查看/终止/重试）
* /api/admin/config 系统配置管理
* /api/admin/audit 审计日志查询
* /api/admin/keys API Key 全局管理
* /api/admin/* 其他系统管理操作
GET /metrics Prometheus 指标（系统监控）

双重验证：Admin 路由同时检查 JWT 中的 admin 角色和请求来源 IP 是否在白名单中。缺一不可。

认证方式速查

路由组 认证方式 Token 来源
Public 无 -
User JWT 登录接口返回，存于 ~/.dove/config.json
Dove API Key 格式 sk_{keyId}_{secret}，注册时生成
Admin JWT + IP 白名单 admin 角色 JWT + 白名单 IP

Server 使用双模认证中间件：自动识别请求携带的是 JWT 还是 API Key，分别走不同的验证逻辑。
