工具开发

工具（Tool）是扩展提供的原子操作，可被 LLM 直接调用。 每个工具完成一件明确的事，具有清晰的输入输出和安全等级声明。

什么是工具

  • 工具是扩展能力的最小执行单元
  • LLM 根据用户意图和工具描述，自主决定调用哪个工具
  • 工具接收结构化参数，返回结构化结果
  • 每个工具必须声明安全等级，系统据此决定是否需要用户确认

文件结构

工具文件放在扩展目录的 tools/ 子目录下，每个文件导出一个工具定义：

我的扩展/
└── tools/
    ├── 查询数据.js
    ├── 写入记录.js
    └── 删除文件.js

系统在加载扩展时自动扫描 tools/ 目录，注册所有工具。

工具定义格式

export default {
  name: '查询数据',
  description: '根据条件查询用户数据，返回匹配的记录列表',
  safetyLevel: 'safe',

  parameters: {
    type: 'object',
    properties: {
      集合名: {
        type: 'string',
        description: '要查询的数据集合名称'
      },
      条件: {
        type: 'object',
        description: '查询过滤条件',
        default: {}
      },
      限制数量: {
        type: 'number',
        description: '最多返回的记录数',
        default: 20,
        maximum: 100
      }
    },
    required: ['集合名']
  },

  execute: async (params, ctx) => {
    const { 集合名, 条件 = {}, 限制数量 = 20 } = params;

    const results = await ctx.db.find('user_data', 集合名, 条件, {
      limit: 限制数量
    });

    return {
      总数: results.length,
      数据: results
    };
  }
};

字段说明

字段 类型 必填 说明
name string 是 工具名称，在扩展内唯一。LLM 通过名称+描述匹配工具
description string 是 工具功能描述。写给 LLM 看的，要清晰说明能做什么、什么时候用
safetyLevel string 是 安全等级：safe | caution | dangerous
parameters object 是 JSON Schema 格式的参数定义
execute async function 是 执行函数，接收 (params, ctx)

安全等级详解

safe — 安全操作

只读操作，无副作用。LLM 可自主调用，无需用户确认。

  • 查询数据（不修改）
  • 格式转换、计算
  • 读取文件内容
  • 获取外部 API 的只读数据
// safe 示例
export default {
  name: '查看天气',
  safetyLevel: 'safe',
  // ...
};

caution — 谨慎操作

会写入或修改数据，但操作可逆。LLM 调用前会提示用户确认。

  • 创建新记录
  • 修改已有数据
  • 发送消息/邮件
  • 写入文件
// caution 示例
export default {
  name: '保存笔记',
  safetyLevel: 'caution',
  // ...
};

dangerous — 危险操作

破坏性或高成本操作，不可逆或很难恢复。必须经过用户明确确认。

  • 删除数据
  • 执行系统命令
  • 大额消费（调用付费 API）
  • 修改权限/配置
// dangerous 示例
export default {
  name: '删除项目',
  safetyLevel: 'dangerous',
  // ...
};

DoveAppContext (ctx)

ctx 是工具与系统交互的唯一通道，提供以下方法：

ctx.db — 数据库操作

// 查询
const docs = await ctx.db.find(db, collection, query, options);
const doc = await ctx.db.findOne(db, collection, query);
const count = await ctx.db.countDocuments(db, collection, query);
const results = await ctx.db.aggregate(db, collection, pipeline);

// 写入
const result = await ctx.db.insertOne(db, collection, doc);
const result = await ctx.db.insertMany(db, collection, docs);

// 更新
const result = await ctx.db.updateOne(db, collection, filter, update);
const result = await ctx.db.updateMany(db, collection, filter, update);

// 删除
const result = await ctx.db.deleteOne(db, collection, filter);
const result = await ctx.db.deleteMany(db, collection, filter);

ctx.storage — 存储操作

// OSS
const content = await ctx.storage.read(path);
await ctx.storage.write(path, content);
await ctx.storage.delete(path);
const files = await ctx.storage.list(prefix);

// 临时文件
const tmpPath = await ctx.storage.createTemp(content, ext);

ctx.fetch — 外部网络请求

// 类似浏览器 fetch API
const response = await ctx.fetch(url, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(data)
});

ctx.events — 事件系统

// 发布事件
await ctx.events.publish('事件名', payload);

// 订阅事件（在 onInit 中使用）
ctx.events.subscribe('事件名', handler);

ctx.extensions — 扩展间调用

// 调用其他扩展的方法
const result = await ctx.extensions.call('扩展名', '方法名', params);

// 读取其他扩展的数据
const data = await ctx.extensions.read('扩展名', 'key');

ctx.log — 日志

ctx.log.info('操作成功', { detail: '...' });
ctx.log.warn('注意', { reason: '...' });
ctx.log.error('操作失败', { error: err });

完整工具示例

/**
 * 工具：创建待办事项
 * 扩展：任务管理
 */
export default {
  name: '创建待办',
  description: '创建一条新的待办事项，设置标题、优先级和截止日期',
  safetyLevel: 'caution',

  parameters: {
    type: 'object',
    properties: {
      标题: {
        type: 'string',
        description: '待办事项的标题',
        minLength: 1,
        maxLength: 200
      },
      优先级: {
        type: 'string',
        enum: ['高', '中', '低'],
        description: '优先级',
        default: '中'
      },
      截止日期: {
        type: 'string',
        format: 'date',
        description: '截止日期，格式 YYYY-MM-DD（可选）'
      },
      标签: {
        type: 'array',
        items: { type: 'string' },
        description: '标签列表（可选）',
        default: []
      }
    },
    required: ['标题']
  },

  execute: async (params, ctx) => {
    const { 标题, 优先级 = '中', 截止日期, 标签 = [] } = params;

    // 参数校验
    if (!标题.trim()) {
      throw new Error('标题不能为空');
    }

    // 构建待办文档
    const 待办 = {
      标题: 标题.trim(),
      优先级,
      标签,
      状态: '未完成',
      创建时间: new Date().toISOString()
    };

    if (截止日期) {
      待办.截止日期 = 截止日期;
    }

    // 写入数据库（ctx 自动注入用户 ID 过滤）
    const result = await ctx.db.insertOne('user_data', '待办事项', 待办);

    ctx.log.info('创建待办成功', { id: result.insertedId, 标题 });

    return {
      success: true,
      id: result.insertedId,
      message: `已创建待办「${标题}」，优先级：${优先级}`
    };
  }
};

最佳实践

遵循以下原则编写高质量工具：

  • 单一职责：一个工具只做一件事。「查询+修改」应拆为两个工具
  • Fail Fast：参数不合法、权限不足、外部服务异常时立即抛出错误，不要默默返回空值
  • 禁止静默失败：不要空 catch，不要返回假数据
  • 描述精确：description 是 LLM 选择工具的依据，要写清楚「什么时候用」「能做什么」「不能做什么」
  • 参数校验：利用 JSON Schema 做第一层校验，execute 内做业务层校验
  • 结构化返回：返回对象而非纯文本，方便 LLM 解析和后续处理
  • 安全等级准确：不确定时选更高等级（宁可多确认，不要漏确认）
  • 日志记录：关键操作用 ctx.log 记录，便于调试和审计

常见错误

// ❌ 错误：直接 import 系统模块
import { MongoClient } from 'mongodb';  // 禁止！

// ✅ 正确：通过 ctx 访问
const data = await ctx.db.find('user_data', '集合', {});

// ❌ 错误：空 catch 吞掉异常
try { await ctx.fetch(url); } catch(e) { return null; }

// ✅ 正确：向上抛出，附带有用信息
try {
  await ctx.fetch(url);
} catch(e) {
  throw new Error(`请求 ${url} 失败: ${e.message}`);
}

// ❌ 错误：硬编码 OSS 路径
await ctx.storage.write('dove/files/xxx.txt', content);

// ✅ 正确：使用声明的路径模式
await ctx.storage.write('用户文件/report.txt', content);
