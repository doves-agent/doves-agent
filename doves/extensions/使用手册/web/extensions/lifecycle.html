生命周期

扩展从加载到运行再到卸载，经历一个完整的生命周期。 理解生命周期有助于在正确的阶段执行初始化逻辑，以及处理异常情况。

加载流程总览

扩展在鸽子启动时由 _loader.js 加载。完整的加载流程包含 14 个步骤：

┌─────────────────────────────────────────────────────┐
│              扩展加载 14 步流水线                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ① 权限声明验证 + DoveAppContext 创建               │
│  ② 开发者身份验证 + 授权检查                         │
│  ③ LLM 层：意图注册                                 │
│  ④ LLM 层：角色注册                                 │
│  ⑤ LLM 层：执行增强                                 │
│  ⑥ 工具层：技能注册                                 │
│  ⑦ 工具层：工具注册                                 │
│  ⑧ 工具层：MCP 配置                                 │
│  ⑨ 能力注册                                        │
│  ⑩ 策略意图注册                                     │
│  ⑪ Web 资源上传          ──┐                        │
│  ⑫ DB 权限注册            ──┤ 并行执行              │
│  ⑬ onInit 钩子执行        ──┘                       │
│  ⑭ 服务启动（如有声明）                              │
│                                                     │
└─────────────────────────────────────────────────────┘

详细步骤说明

步骤 1：权限声明验证 + DoveAppContext 创建

系统解析 manifest.js 中的 permissions 字段，验证格式合法性， 然后基于声明的权限创建该扩展专属的 DoveAppContext 实例。

// 系统内部逻辑（开发者不需要关心实现）
const ctx = new DoveAppContext({
  extensionName: manifest.name,
  permissions: manifest.permissions,
  scope: determineScope(manifest)
});

ctx 是扩展与系统交互的唯一通道。权限验证失败（如格式错误、非法 scope）会在此步骤中止加载。

步骤 2：开发者身份验证 + 授权检查

验证 manifest.developer 中的签名是否有效：

  • 检查开发者 ID 是否已注册
  • 用 hmac-sha256 验证 manifest 内容完整性
  • 确定授权模式：official / production / dev

开发模式（dove app dev）下跳过签名验证，但权限自动降级为 user_scoped。

步骤 3：LLM 层 — 意图注册

加载 intent.js，将扩展的意图关键词和匹配逻辑注册到全局意图路由中。 之后当用户输入消息时，系统通过意图匹配决定激活哪个扩展。

// intent.js 在此步骤被加载并注册
// 注册后，"查天气" 这类输入会被路由到天气查询扩展

步骤 4：LLM 层 — 角色注册

加载 roles.js，注册 LLM 角色定义。鸽子在执行该扩展的任务时会使用对应的角色 prompt。

步骤 5：LLM 层 — 执行增强

加载 execution.js（如有），注册前置/后置处理钩子。 这些钩子在 LLM 调用前后执行，可用于上下文注入、结果后处理等。

步骤 6：工具层 — 技能注册

扫描 skills/ 目录，加载并注册所有技能模块。每个 .js 文件被注册为一个技能。

步骤 7：工具层 — 工具注册

扫描 tools/ 目录，加载并注册所有工具。验证每个工具的定义格式（name、description、safetyLevel、parameters、execute）。

步骤 8：工具层 — MCP 配置

如果扩展声明了 MCP（Model Context Protocol）配置，在此步骤完成 MCP 连接初始化。

步骤 9：能力注册

将扩展的 abilities 关键词注册到全局能力注册表中。 能力管理器通过这些关键词进行能力发现和路由。

步骤 10：策略意图注册

加载 strategy.js，注册执行策略。策略定义了如何编排技能和工具来完成复杂任务。

步骤 11-13：并行执行

以下三个步骤可以并行执行，提高加载速度：

步骤 内容 说明
11 Web 资源上传 将 web/ 目录下的前端资源上传到系统 Web 网关
12 DB 权限注册 在数据库层面注册权限规则，确保运行时查询过滤正确生效
13 onInit 钩子执行 执行扩展的初始化逻辑（如建立连接、预热缓存、事件订阅）

onInit 钩子示例

// 在 manifest.js 中声明
export default {
  // ...
  onInit: async (ctx) => {
    // 订阅需要监听的事件
    ctx.events.subscribe('用户.登录', async (event) => {
      ctx.log.info('用户登录', { userId: event.userId });
    });

    // 预热缓存
    const config = await ctx.db.findOne('extensions', '配置', { name: '天气查询' });
    ctx.cache.set('config', config);

    ctx.log.info('天气查询扩展初始化完成');
  }
};

步骤 14：服务启动

如果扩展声明了长期运行的服务（如定时任务、后台处理器），在此步骤启动。

// manifest.js 中声明服务
export default {
  // ...
  service: {
    type: 'interval',
    interval: 60000,  // 每分钟执行
    handler: async (ctx) => {
      // 定时清理过期缓存
      await ctx.db.deleteMany('extensions', '天气缓存', {
        过期时间: { $lt: new Date() }
      });
    }
  }
};

卸载流程

扩展卸载时执行逆向清理：

┌──────────────────────────────────┐
│          扩展卸载流程             │
├──────────────────────────────────┤
│ 1. 停止服务（如有）              │
│ 2. 执行 onDestroy 钩子          │
│ 3. 取消事件订阅                  │
│ 4. 注销工具和技能                │
│ 5. 注销意图和策略                │
│ 6. 撤销 DB 权限注册             │
│ 7. 清理 Web 资源                │
│ 8. 销毁 DoveAppContext          │
└──────────────────────────────────┘

onDestroy 钩子

export default {
  // ...
  onDestroy: async (ctx) => {
    // 释放外部连接
    // 保存必要状态
    ctx.log.info('扩展正在卸载，执行清理...');
  }
};

错误处理

核心原则：单个扩展加载失败不会阻塞其他扩展和系统启动。

加载失败处理

// _loader.js 内部逻辑
for (const ext of extensions) {
  try {
    await loadExtension(ext);
    log.info(`扩展 [${ext.name}] 加载成功`);
  } catch (error) {
    log.error(`扩展 [${ext.name}] 加载失败`, { error: error.message });
    // 记录失败，继续加载下一个扩展
    failedExtensions.push({ name: ext.name, error: error.message });
  }
}

常见加载失败原因

错误 阶段 处理
权限格式非法 步骤 1 加载中止，输出格式错误详情
签名验证失败 步骤 2 加载中止，可能是 manifest 被篡改
工具定义不合法 步骤 7 跳过该工具，其余工具正常注册
onInit 抛出异常 步骤 13 记录错误，扩展仍标记为已加载（可用但可能功能不完整）
依赖扩展未加载 预处理 延迟加载，所有依赖就绪后重试

热重载

开发模式下支持热重载，修改扩展文件后自动重新加载：

# 启动开发模式（自动监听文件变化）
dove app dev 天气查询 --watch

热重载流程：

  1. 检测到文件变化
  2. 执行卸载流程（上述 8 步）
  3. 重新执行加载流程（14 步）
  4. 输出重载结果

热重载会清除扩展的运行时状态（内存缓存、事件订阅等）。 如果扩展依赖持久状态，应将状态存储在数据库中而非内存。

生命周期钩子总结

钩子 时机 用途
onInit 加载完成后（步骤 13） 初始化连接、订阅事件、预热缓存
onDestroy 卸载时 释放资源、保存状态、清理连接
service.handler 周期性执行（步骤 14 启动） 定时任务、后台处理
