权限体系

白鸽的权限体系确保扩展只能访问其声明的资源，实现多租户隔离和最小权限原则。 所有资源访问通过 DoveAppContext (ctx) 代理，运行时强制执行权限检查。

为什么需要权限

  • 安全性：防止恶意或有缺陷的扩展越权访问系统资源
  • 多租户隔离：确保用户 A 的扩展无法读取用户 B 的数据
  • 透明度：用户安装扩展前可以清楚地看到它需要哪些权限
  • 可审计：所有权限使用都有记录，便于安全审计

四种权限作用域（Scope）

作用域 标识 数据可见范围 典型场景
全局共享 shared 所有数据，无用户过滤 系统配置、公共知识库、全局统计
用户隔离 user_scoped 仅当前用户的数据 用户笔记、个人设置、历史记录
任务隔离 task_scoped 仅当前任务/根任务的数据 任务中间状态、临时缓存
扩展私有 extension 仅本扩展内部可访问 扩展配置、内部缓存、运行状态

作用域示例

// shared — 所有用户可见的公共数据
{
  db: 'system',
  collection: '公告',
  scope: 'shared',
  actions: ['find']
}

// user_scoped — 自动按用户 ID 过滤
{
  db: 'user_data',
  collection: '笔记',
  scope: 'user_scoped',
  userField: '用户ID',      // 系统自动注入过滤条件
  actions: ['find', 'insertOne', 'updateOne', 'deleteOne']
}

// task_scoped — 自动按任务 ID 过滤
{
  db: 'task_data',
  collection: '执行步骤',
  scope: 'task_scoped',
  actions: ['find', 'insertOne']
}

// extension — 扩展私有命名空间
{
  db: 'extensions',
  collection: '天气查询_缓存',
  scope: 'extension',
  actions: ['find', 'insertOne', 'deleteMany']
}

user_scoped 必须声明 userField，系统会在每次查询时自动注入 { [userField]: 当前用户ID } 过滤条件。扩展代码无法绕过此限制。

六种权限类别

1. databases — 数据库访问

声明对 MongoDB 集合的访问权限，精确到操作级别。

databases: [
  {
    db: 'user_data',             // 数据库名
    collection: '任务',          // 集合名
    scope: 'user_scoped',        // 作用域
    userField: '用户ID',         // scope 为 user_scoped 时必填
    actions: [                   // 允许的操作
      'find',
      'findOne',
      'insertOne',
      'insertMany',
      'updateOne',
      'updateMany',
      'deleteOne',
      'deleteMany',
      'aggregate',
      'countDocuments'
    ]
  }
]
操作 说明 安全等级
find查询多条文档读取
findOne查询单条文档读取
countDocuments统计文档数量读取
aggregate聚合管道查询读取
insertOne插入单条写入
insertMany批量插入写入
updateOne更新单条写入
updateMany批量更新写入
deleteOne删除单条危险
deleteMany批量删除危险

2. storage — 存储访问

声明对 OSS 对象存储和 git-storage 的访问权限。

storage: [
  // OSS 存储
  {
    type: 'oss',
    path: '用户文件/${userId}/*',   // 支持变量替换
    actions: ['read', 'write', 'delete', 'list']
  },
  // Git 存储
  {
    type: 'git-storage',
    repo: '知识库',
    actions: ['read', 'write']
  },
  // 记忆存储
  {
    type: 'memory',
    namespace: '天气偏好',
    actions: ['read', 'write']
  }
]

3. apis — 内部 API 访问

声明对系统内部 API 的访问权限。

apis: [
  {
    path: '/api/user/profile',
    actions: ['read']           // read = GET, write = POST/PUT/DELETE
  },
  {
    path: '/api/tasks/*',
    actions: ['read', 'write']
  }
]

4. events — 事件订阅/发布

声明对系统事件的订阅和发布权限。

events: [
  {
    name: '任务.完成',
    actions: ['subscribe']       // 订阅此事件
  },
  {
    name: '天气.告警',
    actions: ['publish']         // 发布此事件
  },
  {
    name: '数据.变更.*',         // 支持通配符
    actions: ['subscribe']
  }
]

5. extensions — 扩展间通信

声明与其他扩展的交互权限。

extensions: [
  {
    name: '通知',
    actions: ['call']            // 调用通知扩展的方法
  },
  {
    name: '数据分析',
    actions: ['read']            // 读取数据分析扩展的公开数据
  },
  {
    name: '文档',
    actions: ['call', 'read', 'write']
  }
]
动作 说明
call调用目标扩展暴露的方法
read读取目标扩展的公开数据
write向目标扩展写入数据

6. externalUrls — 外部网络访问

声明扩展需要访问的外部 URL，这是最敏感的权限类别之一。

externalUrls: [
  {
    url: 'https://api.openai.com/*',
    actions: ['fetch'],              // HTTP 请求
    description: '调用 OpenAI API'   // 必须说明用途
  },
  {
    url: 'wss://realtime.example.com',
    actions: ['connect'],            // WebSocket 连接
    description: '实时数据推送'
  },
  {
    url: 'https://cdn.example.com/*',
    actions: ['iframe'],             // 嵌入 iframe
    description: '加载 CDN 资源'
  }
]
动作 说明
fetchHTTP/HTTPS 请求（GET、POST 等）
connectWebSocket 或持久连接
iframe在 Web 页面中嵌入 iframe

DoveAppContext 权限执行

扩展的所有系统访问都通过 DoveAppContext (ctx) 代理，ctx 在每次操作前检查权限：

// 扩展代码中
export default {
  execute: async (params, ctx) => {
    // ctx.db — 数据库操作（自动注入 scope 过滤）
    const 笔记列表 = await ctx.db.find('user_data', '笔记', { 标签: '重要' });
    // ↑ 如果 scope 是 user_scoped，实际查询会变成:
    // { 标签: '重要', 用户ID: '当前用户ID' }

    // ctx.storage — 存储操作
    const 文件 = await ctx.storage.read('用户文件/report.pdf');

    // ctx.fetch — 外部网络请求（检查 externalUrls 权限）
    const resp = await ctx.fetch('https://api.example.com/data');

    // ctx.events — 事件操作
    await ctx.events.publish('数据.更新', { id: '123' });

    // ctx.extensions — 扩展间调用
    await ctx.extensions.call('通知', '发送', { 内容: '处理完成' });
  }
};

如果扩展尝试访问未声明的权限，ctx 会立即抛出 PermissionDeniedError，包含详细的权限缺失信息。

授权模式

模式 适用场景 权限限制
official 官方扩展 可使用所有 scope，包括 shared
production 已审核的第三方扩展 可使用 user_scoped、task_scoped、extension
dev 开发中的扩展（沙箱） 仅限 user_scoped，且使用 mock 后端

开发模式（dove app dev）下，所有权限自动降级为 user_scoped， 并且后端服务替换为内存 mock。这确保开发阶段不会影响生产数据。

安装确认流程

用户安装扩展时，系统会展示权限摘要：

$ dove app install 天气查询

┌─────────────────────────────────────────────┐
│  扩展：天气查询 v1.0.0                       │
│  作者：dev_a1b2c3d4                         │
│                                             │
│  请求的权限：                                │
│                                             │
│  📂 数据库                                   │
│    · user_data.天气收藏 [读/写/删] (用户隔离) │
│                                             │
│  🌐 外部网络                                 │
│    · api.weatherapi.com [HTTP请求]           │
│      用途：调用天气 API 获取数据              │
│                                             │
│  确认安装？[y/N]                             │
└─────────────────────────────────────────────┘

最佳实践

  • 最小权限：只申请实际需要的权限，不要「预留」
  • 优先 user_scoped：除非确实需要跨用户数据，否则使用 user_scoped
  • 说明用途：externalUrls 必须填写 description，其他类别也建议加注释
  • 粒度适当：按功能拆分权限，避免一个大集合包揽所有数据
  • 避免 deleteMany：除非确实需要批量删除，否则只申请 deleteOne
