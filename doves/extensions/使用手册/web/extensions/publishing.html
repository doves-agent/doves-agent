发布与安装

扩展开发完成后，通过发布流程提交到扩展市场。 用户可以搜索、查看详情、确认权限后安装扩展。

开发者注册

发布扩展前，需要先注册为开发者：

# 注册开发者账号
dove developer register

# 系统会要求提供：
# - 开发者名称
# - 联系邮箱
# - 组织（可选）

# 注册成功后获得：
# - 开发者 ID (dev_xxx)
# - 签名密钥（用于 manifest 签名）

签名密钥只在注册时显示一次，务必安全保存。丢失后需要重新生成（会使已发布扩展的签名失效）。

签名机制

签名用于验证扩展的完整性和开发者身份。发布时系统自动生成：

// manifest.js 中的签名字段
developer: {
  id: 'dev_a1b2c3d4',
  signature: 'hmac-sha256:e3b0c44298fc1c149afbf4c8996fb924...'
}

签名算法：

  1. 提取 manifest 中除 developer.signature 外的所有内容
  2. JSON 序列化（key 排序，无空格）
  3. 使用开发者密钥进行 HMAC-SHA256 签名
  4. 结果写入 developer.signature 字段

签名在 dove app publish 时自动计算和写入，开发者无需手动操作。 系统在加载扩展时验证签名，防止 manifest 被篡改。

发布前验证

dove app validate 执行以下 10 项检查：

# 检查项 说明
1 manifest 格式 必填字段存在、类型正确、version 符合 semver
2 文件引用 所有路径引用的文件/目录实际存在
3 权限声明 格式合法、scope 有效、actions 在允许范围内
4 工具定义 每个工具有 name、description、safetyLevel、parameters、execute
5 技能定义 每个技能有 name、description、execute
6 安全扫描 无直接 import 系统模块、无 eval、无文件系统直接操作
7 依赖检查 声明的依赖扩展存在、无循环依赖
8 开发者身份 developer.id 已注册且有效
9 命名冲突 扩展名和工具名不与已有扩展冲突
10 文件大小 总包大小在限制内（默认 10MB） 
# 执行验证
dove app validate 天气查询

# 输出示例（全部通过）：
# ✓ manifest 格式正确
# ✓ 文件引用完整
# ✓ 权限声明合法
# ✓ 工具定义规范 (3 个工具)
# ✓ 技能定义规范 (2 个技能)
# ✓ 安全扫描通过
# ✓ 依赖检查通过
# ✓ 开发者身份有效
# ✓ 无命名冲突
# ✓ 文件大小: 156KB / 10MB
#
# 验证通过 ✓ 可以发布

发布流程

# 发布到扩展市场
dove app publish 天气查询

发布命令执行以下操作：

  1. 运行完整验证（同 validate）
  2. 计算 manifest 签名
  3. 打包扩展文件
  4. 上传到扩展注册中心
  5. 触发审核流程（如需要）
# 发布成功输出：
# 正在验证... ✓
# 正在签名... ✓
# 正在打包... ✓ (156KB)
# 正在上传... ✓
#
# 发布成功！
# 扩展：天气查询 v1.0.0
# 状态：等待审核
#
# 审核通过后用户即可安装。
# 查看状态：dove app status 天气查询 --market

审核流程

第三方扩展需要经过审核才能上架：

┌───────────┐     ┌──────────┐     ┌───────────┐
│  开发者   │     │  审核员  │     │  扩展市场 │
│           │     │          │     │           │
│  submit ──┼────▶│  review  │     │           │
│           │     │          │     │           │
│           │     │ approve ─┼────▶│  上架     │
│           │     │   or     │     │           │
│           │◀────┼─ reject  │     │           │
│           │     │(附原因)   │     │           │
└───────────┘     └──────────┘     └───────────┘

审核命令（管理员）

# 查看待审核列表
dove app review --list

# 查看具体扩展
dove app review 天气查询 --detail

# 批准
dove app review 天气查询 --approve --comment "审核通过，功能正常"

# 拒绝（必须附原因）
dove app review 天气查询 --reject --comment "externalUrls 中存在未说明用途的域名"

审核标准

  • 权限是否遵循最小权限原则
  • externalUrls 是否有合理说明
  • 无恶意代码或数据泄露风险
  • 功能描述与实际行为一致
  • 代码质量达到基本标准

版本更新

发布更新

# 修改 manifest.js 中的 version
# 1.0.0 → 1.1.0（新功能）或 1.0.1（修复）

# 重新发布
dove app publish 天气查询

# 如果是 minor/patch 更新，自动通过审核
# 如果是 major 更新或权限变更，需要重新审核

用户更新

# 检查更新
dove app update --check

# 更新指定扩展
dove app update 天气查询

# 更新所有扩展
dove app update --all

如果更新版本包含新增权限，系统会再次弹出权限确认。 用户拒绝新权限则保持当前版本不变。

版本管理策略

版本变更 含义 审核要求
1.0.0 → 1.0.1 Bug 修复 自动通过
1.0.0 → 1.1.0 新功能（向后兼容） 自动通过（无权限变更时）
1.0.0 → 2.0.0 破坏性变更 需要审核
任意版本 + 权限新增 权限范围扩大 需要审核 + 用户重新确认
