扩展系统概述

扩展（Extension）是白鸽系统的能力插件，为鸽子（智能体）提供可组合的执行能力。 每个扩展封装一组工具、技能和策略，通过标准化接口接入系统。

设计哲学

鸽子之下的一切都是能力。

鸽子本身只是调度与决策层，真正的执行力来自扩展。 编码能力、文档生成、视频处理、数据分析——所有这些都以扩展形式存在，按需组合。

扩展 vs 应用

概念 性质 说明
扩展（Extension） 技术实体 代码模块，声明权限、注册工具和技能，运行在沙箱中
应用（Application） 用户界面概念 面向用户的入口，可由一个或多个扩展组合而成

一个「应用」可以组合多个扩展的能力，但扩展本身是独立、可复用的技术单元。

能力层级

白鸽系统中的能力按以下层级组织：

能力管理器（Capability Manager）
├── 扩展（Extensions）
│   ├── 技能（Skills）    — 高级执行逻辑
│   └── 工具（Tools）     — 原子操作
├── LLM Providers         — 大模型调用能力
└── MCP（Model Context Protocol）— 外部模型上下文

能力管理器负责在系统启动时发现、验证并注册所有能力。 它扫描 doves/extensions/ 目录，加载每个扩展的 manifest，完成权限验证和资源注册。

核心特征

  • 沙箱隔离：扩展只能通过 DoveAppContext (ctx) 访问系统资源，无法直接操作数据库或发起网络请求
  • 权限声明式：所有资源访问必须在 manifest 中预先声明，运行时由系统强制执行
  • 多租户安全：user_scoped 权限确保扩展只能访问当前用户的数据
  • 热加载：扩展可在运行时加载/卸载，单个扩展的故障不影响系统整体
  • 标准化接口：统一的 manifest 规范、工具定义格式和生命周期钩子
