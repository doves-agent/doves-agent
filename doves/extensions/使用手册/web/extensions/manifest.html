manifest.js 规范

manifest.js 是扩展的声明文件，定义了扩展的身份、能力、权限和资源引用。 系统通过解析 manifest 完成扩展的发现、验证和加载。

完整结构

export default {
  // ═══ 基本信息 ═══
  name: '扩展名称',              // [必填] 唯一标识，中文或英文
  version: '1.0.0',             // [必填] 语义化版本号 (semver)
  description: '扩展功能描述',    // [必填] 一句话描述扩展用途
  abilities: ['能力1', '能力2'], // [必填] 能力关键词，用于意图匹配

  // ═══ 依赖 ═══
  dependencies: ['其他扩展名'],   // 依赖的其他扩展

  // ═══ LLM 层模块 ═══
  intent: './intent.js',         // 意图识别模块
  strategy: './strategy.js',     // 执行策略模块
  roles: './roles.js',           // 角色/人设模块
  review: './review.js',         // 审查标准模块
  execution: './execution.js',   // 执行增强模块
  workflow: './workflow.js',      // 工作流定义

  // ═══ 工具层模块 ═══
  skills: './skills',            // 技能目录路径
  tools: './tools',              // 工具目录路径

  // ═══ 开发者身份 ═══
  developer: {
    id: 'dev_xxx',               // 开发者 ID
    signature: 'hmac-sha256:...' // 签名（发布时生成）
  },

  // ═══ 权限声明 ═══
  permissions: {
    databases: [...],            // 数据库访问
    storage: [...],              // 存储访问
    apis: [...],                 // 内部 API 访问
    events: [...],               // 事件订阅/发布
    extensions: [...],           // 扩展间通信
    externalUrls: [...]          // 外部网络访问
  },

  // ═══ Web 资源 ═══
  web: {
    nav: { ... },                // 导航配置
    pages: { ... }               // 页面路由
  }
};

字段详解

基本信息

字段 类型 必填 说明
name string 是 扩展唯一名称。在系统内全局唯一，建议使用中文描述性名称
version string 是 遵循 semver 规范：主版本.次版本.修订号
description string 是 简要描述扩展功能，会显示在扩展市场和安装确认中
abilities string[] 是 能力关键词数组，用于意图匹配和搜索。例如 ['编码', '代码生成', 'Python']
dependencies string[] 否 依赖的其他扩展名称。系统会确保依赖项先于本扩展加载

LLM 层模块

字段 说明
intent 意图识别模块。定义什么样的用户输入应该激活此扩展。导出一个函数，接收用户消息，返回匹配分数
strategy 执行策略。定义扩展处理任务的编排逻辑：单步直接执行、多步骤流水线、或条件分支
roles 角色定义。为 LLM 提供人设和行为约束，影响输出风格和专业度
review 审查标准。定义输出质量检查规则，用于自动审查生成结果
execution 执行增强。在 LLM 调用前后注入处理逻辑（如上下文补充、结果后处理）
workflow 工作流定义。描述复杂任务的多阶段执行流程

intent.js 示例

export default {
  keywords: ['天气', '气温', '下雨', '预报'],
  examples: [
    '今天北京天气怎么样',
    '明天会下雨吗',
    '查一下上海的温度'
  ],
  // 自定义匹配逻辑（可选）
  match: (message) => {
    if (message.includes('天气') || message.includes('气温')) {
      return { score: 0.9 };
    }
    return { score: 0 };
  }
};

roles.js 示例

export default {
  default: {
    name: '气象专家',
    prompt: '你是一位专业的气象分析师，擅长解读天气数据并给出生活建议。回答简洁实用。'
  }
};

工具层模块

字段 说明
skills 技能目录路径（相对于扩展根目录）。目录下的每个 .js 文件会被自动注册为一个技能
tools 工具目录路径。目录下的每个 .js 文件会被自动注册为一个工具

developer 开发者身份

developer: {
  id: 'dev_a1b2c3d4',           // 开发者注册后获得的唯一 ID
  signature: 'hmac-sha256:e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855'
}

签名在 dove app publish 时自动生成。开发阶段可留空，但发布前必须填写。 签名基于 manifest 内容 + 开发者密钥生成，用于验证扩展完整性。

permissions 权限声明

权限声明是 manifest 中最关键的部分。详细说明请参见 权限体系。

快速示例

permissions: {
  databases: [
    {
      db: 'user_data',
      collection: '笔记',
      scope: 'user_scoped',
      userField: '用户ID',
      actions: ['find', 'insertOne', 'updateOne', 'deleteOne']
    }
  ],
  storage: [
    {
      type: 'oss',
      path: '天气缓存/*',
      actions: ['read', 'write']
    }
  ],
  apis: [
    {
      path: '/api/user/profile',
      actions: ['read']
    }
  ],
  events: [
    {
      name: '天气.更新',
      actions: ['publish']
    }
  ],
  extensions: [
    {
      name: '通知',
      actions: ['call']
    }
  ],
  externalUrls: [
    {
      url: 'https://api.weatherapi.com/*',
      actions: ['fetch']
    }
  ]
}

web 资源配置

如果扩展提供 Web 前端界面：

web: {
  nav: {
    title: '天气查询',
    icon: 'cloud-sun',
    order: 50
  },
  pages: {
    '/weather': './web/index.html',
    '/weather/history': './web/history.html'
  }
}

Web 资源在扩展加载的第 11 步上传到系统，通过统一的 Web 网关提供服务。 页面可使用系统提供的 UI 组件库。

最小化示例

一个最简单的合法 manifest：

export default {
  name: '问候',
  version: '1.0.0',
  description: '简单的问候扩展示例',
  abilities: ['问候', '打招呼'],
  tools: './tools',
  permissions: {}
};

验证规则

dove app validate 会检查以下规则：

  • name、version、description、abilities 必须存在且非空
  • version 必须符合 semver 格式
  • 所有路径引用的文件/目录必须实际存在
  • 权限声明格式正确、scope 合法
  • 工具定义符合 JSON Schema 规范
  • 无循环依赖
