LLM 提供商

白鸽系统通过统一的 OpenAI 兼容 API 客户端（提供商客户端）接入多家 LLM 提供商。只有 Doves 调用 LLM，Server 和 CLI 不调用。

架构约束：LLM 调用是 Doves 的独占能力。Server 和 CLI 永远不直接调用 LLM。

三种调用模式

调用（非流式）

一次性返回完整响应。用于短回复、工具调用判断等场景。

流式调用

逐 token 流式返回。用于长文本生成、实时展示进度。

工具调用

带 function calling 的调用。LLM 返回 tool_calls，由执行器处理。KISS 执行循环的核心。

模型角色与选择

系统定义 5 种模型角色，每种角色对应不同的使用场景：

角色 用途 典型模型
推理（Reasoning） 复杂推理、深度分析 DeepSeek-v4-pro, Qwen3-max
Flash（快速） 简单任务、低延迟响应 DeepSeek-v4-flash, Qwen3-mini
视觉（Vision） 图像理解、OCR glm-4v, Qwen3-vl
意图（Intent） 意图识别、分类 Flash 级模型
规划（Planning） 任务分解、方案制定 Reasoning 级模型

KeyManager

KeyManager 负责管理多个提供商的多个 API Key：

Key 轮换

同一提供商多个 Key 轮流使用，避免单 Key 达到速率限制。

配额追踪

追踪每个 Key 的使用量和剩余配额，接近上限时自动切换。

熔断器（Circuit Breaker）

熔断机制：

正常状态（Closed）
  │
  │ 连续失败达到阈值
  ▼
熔断状态（Open）
  │ - 所有请求直接拒绝
  │ - 自动切换到备选提供商
  │
  │ 冷却时间到达
  ▼
半开状态（Half-Open）
  │ - 放行少量试探请求
  │
  ├─ 成功 → 恢复正常（Closed）
  └─ 失败 → 重回熔断（Open）

熔断器按提供商粒度工作。某个提供商熔断后，系统自动降级到其他可用提供商，不影响任务执行。

自动重试

参数 值 说明
最大重试次数 3 首次失败后最多重试 3 次
基础延迟 1 秒 第一次重试前等待 1 秒
退避策略 指数退避 1s → 2s → 4s
可重试错误 网络超时、5xx、429 4xx（非 429）不重试

Token 统计

系统追踪每次 LLM 调用的 token 消耗：

统计维度 说明
任务级 每个任务累计消耗的 input/output tokens
鸽子级 每只鸽子的总消耗，用于计费
提供商级 按提供商统计，辅助配额管理
模型级 按具体模型统计，辅助成本分析
