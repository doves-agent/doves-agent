Server 数据网关

Server 是白鸽系统的纯数据网关，职责边界清晰：认证、数据代理、任务入队。不调用 LLM，不推送任务，不 import Doves 代码。

核心定位：Server 只做数据的搬运工，不理解任务内容，不感知 Doves 执行逻辑。

端口与协议

项目 说明
对外端口 3003（唯一）
对外协议 Noise NX 加密 TCP
内部 Express 绑定在随机 localhost 端口，外部不可访问
明文 HTTP 无，不暴露任何明文端口

启动链

启动流程：
1. 验证密钥（JWT_SECRET, HASH_SECRET, OFFICIAL_DEV_SIGNING_KEY）
   └─ 不满足要求 → process.exit(1)
2. 连接 MongoDB（doves_admin + doves_user_data）
3. 创建索引（确保查询性能）
4. 启动事件调度器（定时任务 + Change Stream）
5. 绑定 Noise NX 加密 TCP 端口 3003
   └─ 开始接受连接

启动硬性要求：JWT_SECRET >= 32 字符，HASH_SECRET >= 32 字符，OFFICIAL_DEV_SIGNING_KEY >= 40 字符。任一不满足，进程拒绝启动。

路由分类

类别 认证要求 说明
Public 无 健康检查、协议文档、注册/登录
Authenticated JWT Token 用户业务操作（DB代理、文件、任务管理等）
Dove API Key 鸽子认领任务、提交结果、心跳
Admin JWT + admin 角色 + IP 白名单 系统管理、指标监控
High-Performance Direct 视接口而定 绕过通用中间件的高性能直连路径

数据库代理

Server 提供统一的 MongoDB 代理接口（/db/*），实现多租户数据隔离。

白名单机制

只允许访问预定义的集合和操作类型。未在白名单中的请求直接拒绝。

自动注入 userId

所有查询自动注入当前用户 ID，确保用户只能访问自己的数据，实现多租户隔离。

IM 通道

Server 支持多个即时通讯平台接入，功能与 CLI 等价。

平台 Webhook 路径 说明
微信 /api/im/wechat/webhook 消息收发、媒体处理
钉钉 /api/im/dingtalk/webhook 消息收发
飞书 /api/im/feishu/webhook 消息收发

IM 与 CLI 的区别仅为：来源标记不同（source: "wechat" vs source: "cli"），以及 IM 通道支持 Server 主动推送通知。

事件调度器

Server 内置事件调度系统，支持多种触发方式：

触发类型 说明 示例
cron 定时触发 每天 3:00 执行任务归档
data_change MongoDB Change Stream 触发 任务状态变更时触发通知
external 外部系统调用触发 Webhook 回调
semantic 语义判断触发（LLM 判定） 条件满足时自动触发

安全机制

JWT 验证

用户请求携带 JWT Token，Server 验证签名和有效期（7天）。

API Key 认证

Doves 使用 API Key（格式 sk_{keyId}_{secret}），Server 用 bcrypt 验证。

登录锁定

5 次失败尝试 → 锁定 30 分钟。防止暴力破解。

速率限制

API 默认 100 req/min/IP，登录/注册有独立更严格的限制。

审计日志

Server 记录所有关键操作的审计日志，存储在 MongoDB 中，TTL 90 天自动清理。

审计日志结构：
{
  时间戳,
  用户ID,
  操作类型,
  操作目标,
  请求来源IP,
  操作结果,
  详情
}

TTL: 90 天后 MongoDB 自动删除

职责边界总结

Server 做的 Server 不做的
认证（JWT + API Key） 调用 LLM
数据代理（MongoDB + OSS） 推送任务给 Doves
任务入队 import Doves 代码
IM Webhook 接入 理解任务内容
事件调度 执行任务逻辑
审计日志 暴露明文 HTTP
