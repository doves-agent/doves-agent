Doves 执行引擎

Doves 是白鸽系统的执行核心。它采用 Flock（鸽群）架构，单进程管理多个鸽子实例，自主轮询任务并调用 LLM 完成执行。

共享 vs 独占资源

类型 资源 说明
共享 DovesProxy 与 Server 的加密 TCP 通信通道，所有鸽子复用
共享 KeyManager API Key 管理、轮换、配额追踪
共享 LLMCaller LLM 调用客户端，含熔断和重试逻辑
共享 ToolRegistry 工具注册表，工具定义全局共享
独占 任务队列 每只鸽子独立的任务处理队列
独占 Token 统计 每只鸽子独立追踪 token 消耗
独占 执行器 KISS 执行循环，互不干扰

KISS 执行器

执行器是 Doves 的核心循环，极致简洁：

KISS 执行循环：

┌─────────────────────────────────────────────────┐
│                                                  │
│  1. 构建消息（系统提示 + 历史 + 用户输入）       │
│     │                                            │
│     ▼                                            │
│  2. callLLM(messages, tools)                     │
│     │                                            │
│     ▼                                            │
│  3. 响应是否包含 tool_calls?                     │
│     │                                            │
│     ├─ 否 → 返回最终回复，任务完成               │
│     │                                            │
│     └─ 是 → 执行工具调用                         │
│            │                                     │
│            ▼                                     │
│         将工具结果追加到消息                      │
│            │                                     │
│            ▼                                     │
│         回到步骤 2（下一轮）                      │
│                                                  │
│  最大轮数：20（DOVE_MAX_TOOL_ROUNDS 可配置）     │
└─────────────────────────────────────────────────┘

极致 KISS：执行器不维护复杂状态机，就是 build → call → execute → repeat 的简单循环。20 轮足以完成绝大多数任务。

工具系统

基础工具集（Fallback）

每次执行始终可用的基础工具：

工具 说明
read_file 读取文件内容
shell_exec 执行 Shell 命令
notify_user 向用户发送通知
request_upload 请求用户上传文件
load_tool_group 动态加载工具组

动态工具加载

通过 load_tool_group 按需加载工具组，避免初始 prompt 过大：

工具组 包含工具数 用途
文件操作 ~3 写入、移动、删除文件
网络请求 ~2 HTTP 请求、下载
数据处理 ~3 JSON/CSV 解析、数据转换
知识管理 ~2 向量记忆读写
代码执行 ~2 沙箱代码运行
...共 13 个工具组，约 29 个工具

LLM 在执行过程中根据需要自主调用 load_tool_group 加载额外工具。一次加载后当次任务内持续可用。

监工（Supervisor）

基于执行轨迹的异常检测系统，防止鸽子陷入死循环或卡死：

卡死检测

30 分钟无实质性进展（无新工具调用、无输出变化）→ 触发干预。

工具循环检测

连续 8 次以上调用同一工具 → 判定为循环，强制中断或切换策略。

时长限制

子任务最长 60 分钟，分支最长 90 分钟。超时自动终止。

优雅关闭

收到 SIGTERM / SIGINT:
  1. 停止接受新任务
  2. 等待当前执行中的任务完成（或达到超时）
  3. 保存检查点（checkpoint）
  4. 释放未完成任务回队列
  5. 断开与 Server 的连接
  6. 进程退出

DovesProxy

DovesProxy 是 Doves 与 Server 通信的唯一通道，封装了所有网络细节：

功能 说明
加密传输 Noise NX 协议，自动握手和密钥协商
自动重连 连接断开后指数退避重连
请求复用 多个鸽子实例共享同一连接池
心跳维护 定期发送心跳保持连接活性

执行限制配置

参数 默认值 环境变量 说明
最大工具轮数 20 DOVE_MAX_TOOL_ROUNDS 单次任务最大 LLM 调用轮数
子任务超时 60 分钟 - 单个子任务的最长执行时间
分支超时 90 分钟 - 任务分支的最长执行时间
卡死阈值 30 分钟 - 无进展持续时长触发干预
循环检测阈值 8 次 - 连续调用同一工具的次数上限
