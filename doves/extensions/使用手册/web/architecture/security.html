安全体系

白鸽系统从设计之初就将安全作为第一优先级。全链路加密、零信任通信、多层认证、数据隔离构成完整的安全纵深。

全链路加密

Noise NX 协议

所有组件间通信（CLI↔Server、Doves↔Server、CLI↔Doves）统一使用 Noise NX 加密 TCP。

零明文原则

系统不存在任何明文 HTTP 端口。不支持协议降级，无 TLS fallback。

单端口暴露

Server 仅暴露 3003 端口。内部 Express 绑定 localhost 随机端口，外部不可达。

加密通信拓扑：

CLI ═══[Noise NX]═══▶ Server:3003 ◀═══[Noise NX]═══ Doves
 │                         │
 │   [Noise NX]            │ (内部 Express: 127.0.0.1:random)
 │                         │ (外部不可访问)
 └─────────────────────────┼──▶ Doves（同账号直连）
                           │
                    无明文端口暴露

硬性约束：无明文 HTTP 端口、无协议降级、无例外。违反此规则的代码不得合入。

双模认证

JWT 认证（用户）

项目 说明
获取方式 登录接口返回
有效期 7 天
存储位置 ~/.dove/config.json
携带方式 请求头中传递
签名算法 HMAC-SHA256（JWT_SECRET）

API Key 认证（Doves）

项目 说明
格式 sk_{keyId}_{secret}
存储 数据库中存 bcrypt 哈希，明文不落盘
验证 keyId 查询记录 → bcrypt.compare(secret, hash)
生命周期 手动创建/吊销，无自动过期

启动密钥验证

Server 启动时强制检查以下密钥，不满足要求进程直接退出：

启动检查：
  JWT_SECRET          >= 32 字符  │ 用于 JWT 签名
  HASH_SECRET         >= 32 字符  │ 用于数据哈希
  OFFICIAL_DEV_SIGNING_KEY >= 40 字符  │ 用于官方扩展签名验证

  任一不满足 → console.error → process.exit(1)
  绝不降级运行

Fail Fast：密钥不合格时进程立即退出，不存在"弱密钥也能运行"的降级模式。

密码策略

规则 要求
最小长度 >= 8 个字符
字符要求 必须包含字母 + 数字
弱密码检测 拒绝常见弱密码（如 12345678、password 等）

登录保护

暴力破解防护：

连续失败计数器（按 IP + 用户名）：
  第 1~4 次失败：返回错误，计数 +1
  第 5 次失败：
    → 锁定该账号 30 分钟
    → 后续请求直接拒绝（不验证密码）
    → 30 分钟后自动解锁，计数器归零

速率限制

接口类别 限制 粒度
通用 API 100 次/分钟 IP
登录接口 更严格的独立限制 IP + 用户名
注册接口 更严格的独立限制 IP

多租户数据隔离

数据隔离机制：

用户请求 → JWT 解析出 userId
         → Server 中间件自动注入 userId 到查询条件
         → MongoDB 查询: { ...userQuery, userId: extractedUserId }
         → 结果: 用户只能看到自己的数据

┌────────────┐     ┌──────────────┐     ┌──────────────┐
│ 用户 A 请求 │────▶│ 自动注入     │────▶│ 查询条件:    │
│            │     │ userId: "A"  │     │ { userId: A }│
└────────────┘     └──────────────┘     └──────────────┘

┌────────────┐     ┌──────────────┐     ┌──────────────┐
│ 用户 B 请求 │────▶│ 自动注入     │────▶│ 查询条件:    │
│            │     │ userId: "B"  │     │ { userId: B }│
└────────────┘     └──────────────┘     └──────────────┘

用户 A 永远看不到用户 B 的数据，无论前端传什么查询条件。

扩展权限沙箱

扩展（Extensions）运行在受限环境中，只能通过 DoveAppContext 访问系统资源：

DoveAppContext

扩展唯一的系统访问入口。通过它获取文件访问、网络请求、数据存储等能力，每项需显式授权。

权限声明

扩展在 manifest 中声明所需权限，安装时用户确认。未声明的能力调用会被拒绝。

Admin IP 白名单

管理员路由（/api/admin/*）除需要 admin 角色的 JWT 外，还检查请求来源 IP 是否在白名单中。双重验证，防止 Token 泄露后的越权操作。

审计日志

字段 说明
时间戳 操作发生的精确时间
用户ID 操作发起者
操作类型 如 login、create_task、delete_key 等
操作目标 被操作的资源标识
请求来源 IP 发起请求的 IP 地址
操作结果 成功/失败
详情 额外上下文信息

审计日志设置 MongoDB TTL 索引，90 天后自动清理。确保存储不会无限增长。

安全层次总结

安全纵深（由外到内）：

Layer 1: 传输加密（Noise NX）
  └─ 防窃听、防篡改、防重放

Layer 2: 认证（JWT / API Key）
  └─ 身份验证、权限区分

Layer 3: 访问控制（角色 + IP 白名单）
  └─ admin 路由双重验证

Layer 4: 数据隔离（userId 自动注入）
  └─ 多租户隔离

Layer 5: 速率限制 + 登录保护
  └─ 防暴力破解

Layer 6: 扩展沙箱（DoveAppContext）
  └─ 第三方代码权限控制

Layer 7: 审计追踪（90天日志）
  └─ 事后追溯
