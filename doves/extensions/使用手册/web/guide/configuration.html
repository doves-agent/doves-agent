配置指南

白鸽系统的配置分为 CLI 客户端配置和 Server 环境变量两部分。本文档涵盖所有可调选项。

CLI 配置文件

CLI 配置文件位于用户主目录下：

~/.dove/config.json

权限要求：该文件包含敏感信息（网关地址、Token），权限必须设置为 600（仅所有者可读写）。

默认配置结构

{
  "gateway": "http://localhost:3003",
  "gateways": [],
  "timeout": 30000,
  "chat": {
    "continuousMode": true
  }
}
字段 类型 说明
gateway string 当前活跃的 Server 网关地址
gateways string[] 多网关列表（Fan-out 模式）
timeout number 请求超时时间（毫秒）
chat.continuousMode boolean 是否默认开启连续对话模式

dove config 命令

通过 CLI 管理配置项，无需手动编辑文件：

命令 说明 示例
dove config list 列出所有配置项 -
dove config get <key> 获取指定配置值 dove config get gateway
dove config set <key> <value> 设置配置项 dove config set timeout 60000
dove config reset 重置为默认配置 -

网关管理

配置 CLI 连接的 Server 网关地址：

# 设置主网关
dove config gateway http://my-server:3003

# 添加备用网关（Fan-out 模式）
dove config gateways add http://backup-server:3003

# 移除备用网关
dove config gateways remove http://backup-server:3003

# 查看所有已配置网关
dove config gateways list

Fan-out 模式：配置多网关后，CLI 可将任务同时分发到多个 Server 实例，适用于多区域部署或负载分散场景。

API Key 管理

白鸽支持多个 LLM 提供商，每个用户可独立配置自己的 API Key：

支持的提供商

提供商 标识符 说明
百炼 BAILIAN 阿里云百炼平台
DeepSeek DEEPSEEK DeepSeek API
智谱 GLM 智谱 GLM 系列模型

Key 操作命令

# 设置 API Key
dove config set-key DEEPSEEK sk-xxxxxxxxxxxxxxxxxxxxxxxx

# 查看已配置的 Key（脱敏显示）
dove config list-keys

# 测试 Key 连通性
dove config test-key DEEPSEEK

提示：test-key 命令会向对应提供商发送一个最小化请求以验证 Key 的有效性和网络连通性。

模型配置

白鸽系统定义了 5 种模型角色，每种角色负责不同类型的任务：

角色 用途 配置命令
推理模型 核心推理与复杂任务执行 dove config reasoning-model
Flash模型 快速响应、简单任务 dove config flash-model
视觉模型 图像理解与视觉任务 dove config vision-model
意图模型 用户意图识别与路由 dove config intent-model
规划模型 任务分解与执行规划 dove config planning-model

配置模型示例

# 设置意图模型为 DeepSeek 的 deepseek-chat
dove config intent-model -p DEEPSEEK -m deepseek-chat

# 设置推理模型为百炼的 qwen-max
dove config reasoning-model -p BAILIAN -m qwen-max

# 设置视觉模型为智谱的 glm-4v
dove config vision-model -p GLM -m glm-4v

模型列表管理

# 列出所有可用模型
dove model list

# 从提供商刷新模型列表
dove model refresh

# 按提供商筛选
dove model list --provider DEEPSEEK

环境变量

以下环境变量通过 .env 文件或系统环境配置，作用于 Server 和 Doves 进程：

变量名 用途 默认值
SERVER_URL Server 监听地址 http://0.0.0.0:3003
MONGODB_URI MongoDB 连接字符串 -
JWT_SECRET Token 签名密钥（>=32 字符） -
HASH_SECRET 密码哈希盐值（>=32 字符） -
DEFAULT_MODEL 默认推理模型 -
FAST_MODEL Flash 模型 -
VISION_MODEL 视觉模型 -
MAX_CONCURRENCY 最大子任务并发数 3
DOVE_MAX_TOOL_ROUNDS 单任务最大工具调用轮数 20
DOVE_DIRECT_PORT CLI 直连 Doves 端口 -
MACHINE_ID 机器唯一标识符 -
OSS_PREFIX OSS 存储路径前缀 dove

必填项：MONGODB_URI、JWT_SECRET、HASH_SECRET 为必填配置，缺少任一项服务将无法启动。

执行档案 (Profile)

执行档案允许为不同场景预设一组参数配置（模型选择、并发度、超时等）：

# 列出所有档案
dove profile list

# 查看档案详情
dove profile show 高质量写作

# 创建新档案
dove profile create

# 使用指定档案执行任务
dove chat --profile 高质量写作 "写一篇关于AI的深度文章"

创建档案时，向导将引导你配置：

  • 模型角色绑定（推理/Flash/视觉等使用哪个模型）
  • 并发度与超时设置
  • 工具调用轮数上限
  • 默认系统提示词

多网关 Fan-out 配置

在多区域部署或需要负载分散时，可配置多个网关实现 Fan-out：

# 添加多个网关
dove config gateways add http://server-beijing:3003
dove config gateways add http://server-shanghai:3003
dove config gateways add http://server-guangzhou:3003

# 查看网关列表
dove config gateways list

Fan-out 模式下的行为：

  • 任务提交 — 自动选择延迟最低的网关
  • 故障转移 — 主网关不可用时自动切换到备用
  • 状态聚合 — dove status 汇总所有网关的状态

注意：多网关模式要求所有 Server 实例连接到同一 MongoDB 集群（或配置 MongoDB 副本集），以保证任务状态一致性。

配置优先级

配置项的生效优先级从高到低：

  1. 命令行参数（--timeout 60000）
  2. 环境变量（DOVE_TIMEOUT=60000）
  3. CLI 配置文件（~/.dove/config.json）
  4. 内置默认值
