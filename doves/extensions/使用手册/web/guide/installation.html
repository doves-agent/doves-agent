安装部署

本文档详细介绍白鸽系统的部署方案，涵盖 Docker 和裸机两种方式，以及各操作系统的注意事项。

系统要求

项目 最低要求 推荐配置
操作系统 Linux / macOS / Windows 10+ Ubuntu 22.04 LTS / Debian 12
Node.js 18.0+ 20 LTS
MongoDB 6.0+ 7.0+
内存 (RAM) 2 GB 4 GB+
磁盘空间 1 GB 10 GB+（含日志与缓存）
网络 可访问 LLM API 端点 稳定公网 / 内网穿透

Docker 部署

Docker 部署是最简单的方式，适合快速体验和生产环境。

前置条件

  • Docker Engine 20.10+
  • Docker Compose V2

部署步骤

# 克隆仓库
git clone https://github.com/anthropic/doves.git
cd doves

# 一键部署（含 MongoDB）
./install.sh --docker --with-mongo --port 3003

Docker Compose 将自动启动以下服务：

  • Server — 数据网关，监听加密 TCP 端口
  • Doves — 任务执行引擎
  • MongoDB — 数据存储

自定义选项

参数 说明 示例
--with-mongo 同时部署内置 MongoDB ./install.sh --docker --with-mongo
--skip-mongo 使用外部 MongoDB 实例 ./install.sh --docker --skip-mongo
--port <n> 指定 Server 监听端口 ./install.sh --docker --port 3100

外部 MongoDB：使用 --skip-mongo 时需在 .env 中配置 MONGODB_URI 指向已有实例。

裸机部署

适合需要完全控制运行环境的场景。

前置条件

  • Node.js 18+（推荐通过 nvm 管理）
  • PM2 进程管理器：npm install -g pm2
  • 可访问的 MongoDB 实例

部署步骤

# 1. 克隆仓库
git clone https://github.com/anthropic/doves.git
cd doves

# 2. 安装依赖
cd 白鸽系统
npm run 安装依赖

# 3. 创建环境配置
cp .env.example .env
# 编辑 .env 填入必要配置（见下方）

# 4. 通过 PM2 启动
pm2 start ecosystem.config.cjs

PM2 生态文件

项目内置 ecosystem.config.cjs，定义了 Server 和 Doves 两个进程：

module.exports = {
  apps: [
    {
      name: 'dove-server',
      script: '白鸽系统/start.js',
      args: '--server',
      instances: 1,
      autorestart: true,
      max_memory_restart: '1G'
    },
    {
      name: 'dove-agent',
      script: '白鸽系统/start.js',
      args: '--doves',
      instances: 1,
      autorestart: true,
      max_memory_restart: '2G'
    }
  ]
};

手动启动（调试用）

node 白鸽系统/start.js

注意：手动启动不具备自动重启能力，仅建议在开发调试时使用。生产环境请使用 PM2。

Windows 部署

Windows 环境有以下额外步骤：

注册系统 PATH

dove setup

该命令将 dove 可执行文件路径写入系统环境变量，重启终端后生效。

使用启动脚本

Windows 下推荐使用内置的批处理启动器：

dove.bat

说明：dove.bat 会自动检测 Node.js 路径并设置必要的环境变量。

环境配置文件 (.env)

最小化配置示例：

SERVER_URL=http://0.0.0.0:3003
MONGODB_URI=mongodb://localhost:27017
JWT_SECRET=<random-32-chars>
HASH_SECRET=<random-32-chars>
OFFICIAL_DEV_SIGNING_KEY=<random-40-chars>

安全提示：JWT_SECRET 和 HASH_SECRET 必须使用强随机字符串（至少 32 字符）。可通过 openssl rand -hex 16 生成。OFFICIAL_DEV_SIGNING_KEY 用于扩展签名验证，至少 40 字符。

服务管理

白鸽 CLI 内置了完整的服务生命周期管理命令：

命令 说明
dove service start 启动 Server + Doves
dove service stop 停止所有服务
dove service restart 重启所有服务
dove service status 查看服务运行状态
dove service logs 查看实时日志输出

开机自启

# 启用开机自启
dove service startup --enable

# 禁用开机自启
dove service startup --disable

该命令会通过 PM2 的 startup 机制注册系统服务。

健康检查

部署完成后，使用以下命令验证系统状态：

# 快速连通性测试
dove ping

# 详细状态报告
dove status

dove ping 测试 CLI 到 Server 的加密通道连通性；dove status 返回各组件的运行状态、版本和资源使用情况。

故障排查：如果 dove ping 失败，请检查：1) Server 进程是否运行；2) 端口是否被占用；3) 防火墙规则是否放行。
