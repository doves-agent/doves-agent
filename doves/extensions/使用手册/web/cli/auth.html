认证命令

认证命令管理用户登录、系统初始化和状态检查。这些命令大部分无需登录即可执行。

dove login

用户登录认证，支持多种认证模式。

dove login [options]

选项

Flag Description Default
-a, --anonymous 匿名登录（受限功能） false
-r, --register 注册新账号并登录 false
--admin 管理员登录 false
-v, --verify 验证当前登录状态 false
--refresh 刷新认证 Token false
-g, --gateway <url> 指定网关地址 默认网关
-u, --username <name> 用户名 交互输入
-p, --password <pwd> 密码 交互输入

示例

# 交互式登录
dove login

# 指定用户名密码登录
dove login -u admin -p mypassword

# 注册新账号
dove login -r

# 匿名登录
dove login -a

# 管理员登录
dove login --admin

# 验证登录状态
dove login -v

# 刷新 Token
dove login --refresh

# 通过指定网关登录
dove login -g "tcp://gateway.example.com:3003"

dove logout

退出当前登录会话，清除本地认证凭据。

dove logout [options]

选项

Flag Description Default
-f, --force 强制退出（即使无法通知 Server） false

示例

# 正常退出
dove logout

# 强制退出（网络不可达时使用）
dove logout -f

dove setup

系统安装/卸载命令，配置环境依赖和系统服务。

dove setup [options]

选项

Flag Description Default
-u, --uninstall 卸载白鸽系统 false
-s, --status 查看安装状态 false

示例

# 安装白鸽系统
dove setup

# 查看安装状态
dove setup -s

# 卸载
dove setup -u

dove init

三步初始化向导，引导用户完成 Server 地址配置、账号设置和 LLM 密钥配置。

dove init

初始化三步骤：

  1. Server URL - 配置 Server 连接地址（默认 tcp://localhost:3003）
  2. 账号 - 登录或注册账号
  3. LLM 密钥 - 配置至少一个 LLM 提供商的 API 密钥

示例

# 运行初始化向导
dove init

# 典型输出：
# 步骤 1/3：Server 地址
# ? 请输入 Server 地址 (tcp://localhost:3003): ▌
#
# 步骤 2/3：账号配置
# ? 登录还是注册？ (登录/注册): ▌
#
# 步骤 3/3：LLM 密钥
# ? 请选择提供商 (Anthropic/OpenAI/DeepSeek): ▌
# ? 请输入 API Key: ▌
#
# ✓ 初始化完成！

dove status

系统状态检查命令，查看连接状态、认证信息和服务健康。

dove status [options]

选项

Flag Description Default
-j, --json JSON 格式输出 false
-g, --gateway 显示网关信息 false
-a, --all 显示全部信息 false
--uid 仅显示用户 ID false
--admin 显示管理员信息 false
--secret 显示敏感信息（密钥完整值） false

子命令

Subcommand Description
ping 测试 Server 连通性
test 测试全链路（Server + LLM）
info 显示系统详细信息
show 显示当前配置概览

示例

# 查看状态
dove status

# 完整状态信息
dove status -a

# JSON 格式（监控脚本用）
dove status -j

# 仅获取 UID
dove status --uid

# 查看网关信息
dove status -g

# 测试全链路
dove status test

# 查看系统详细信息
dove status info

dove ping

连通性测试命令，测量 Server 响应延迟。

dove ping [options]

选项

Flag Description Default
-c, --count <n> 发送次数 3

示例

# 默认 ping 3 次
dove ping

# 输出示例：
# PING tcp://localhost:3003
# 回复: 序号=1 时间=12ms
# 回复: 序号=2 时间=8ms
# 回复: 序号=3 时间=9ms
# --- 统计 ---
# 3 次发送, 3 次接收, 0% 丢包
# 延迟 最小/平均/最大 = 8/9.7/12 ms

# 发送 10 次
dove ping -c 10

权限矩阵

不同角色对系统功能的访问权限：

功能 匿名 普通用户 管理员
ping / status 可 可 可
login / logout 可 可 可
setup / init 可 可 可
chat（基本） 受限 可 可
chat（高级选项） 不可 可 可
task 管理 不可 自己的 全部
dove 注册/删除 不可 可 可
config 修改 不可 自己的 全局
storage 管理 不可 自己的配额 全部
memory 访问 不可 自己的 全部
service 管理 查看 查看 全部
用户管理 不可 不可 可
app 审核/发布 不可 不可 可

安全提示：管理员登录建议仅在受信任的终端环境中使用，避免在共享机器上保留管理员会话。
