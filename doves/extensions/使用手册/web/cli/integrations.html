集成命令

集成命令管理白鸽系统与外部服务的对接，包括微信、事件调度、通知、团队和扩展应用。

dove wechat

微信集成管理命令（别名：wx）。绑定微信账号，启用消息监听和自动回复。

dove wechat [action] [options]

Actions

Action Description
bind / login 绑定微信（扫码登录）
unbind / logout 解绑微信
status / info 查看绑定状态
test 发送测试消息
listen 启动消息监听
enable / on 启用微信通道
disable / off 禁用微信通道

选项

Flag Description Default
--json JSON 格式输出 false

示例

# 绑定微信（弹出二维码）
dove wechat bind

# 查看绑定状态
dove wx status

# 发送测试消息
dove wx test

# 启动消息监听
dove wx listen

# 启用微信通道
dove wx on

# 禁用微信通道
dove wx off

# 解绑微信
dove wx unbind

微信通道与 CLI 功能等价，区别仅为来源标记（source: "wechat"）和双向推送能力。

dove event

事件调度管理命令（别名：evt）。创建定时任务、语义事件和外部触发器。

dove event [action] [name] [options]

Actions

Action Description
schedule <name> 创建定时事件
semantic <name> 创建语义事件
list / ls 列出所有事件
show <name> 查看事件详情
trigger <name> 手动触发事件
check 检查待触发事件
delete <name> 删除事件
add-handler <name> 添加事件处理器
remove-handler <name> 移除事件处理器
disable <name> 禁用事件
enable <name> 启用事件
quota 查看事件配额

选项

Flag Description Default
--cron <expr> Cron 表达式 -
--condition <expr> 触发条件表达式 -
--task <desc> 触发时执行的任务描述 -
--threshold <n> 语义匹配阈值 0.8
--cooldown <seconds> 冷却时间（秒） 60
--no-llm-confirm 触发时无需 LLM 二次确认 false
-t, --type <type> 事件类型：scheduled, data_change, semantic, external, intent_driven -
--action-text <text> 处理器动作文本 -
--handler-id <id> 处理器 ID -
--json JSON 格式输出 false

事件类型说明

  • scheduled - 定时事件，使用 Cron 表达式
  • data_change - 数据变更事件，监听数据源变化
  • semantic - 语义事件，当对话匹配语义条件时触发
  • external - 外部事件，由外部系统触发
  • intent_driven - 意图驱动，当检测到特定用户意图时触发

示例

# 创建定时事件（每天早上 9 点）
dove event schedule "日报提醒" --cron "0 9 * * *" --task "提醒我写日报"

# 创建语义事件
dove event semantic "紧急求助" --condition "用户表示遇到紧急问题" --threshold 0.85 --task "立即响应并提供帮助"

# 列出所有事件
dove evt ls

# 按类型过滤
dove evt ls -t scheduled

# 手动触发事件
dove evt trigger "日报提醒"

# 查看事件详情
dove evt show "日报提醒"

# 添加处理器
dove evt add-handler "紧急求助" --action-text "发送微信通知"

# 移除处理器
dove evt remove-handler "紧急求助" --handler-id handler_123

# 禁用事件
dove evt disable "日报提醒"

# 查看配额
dove evt quota

# 删除事件
dove evt delete "旧事件"

dove notify

通知管理命令（别名：ntf）。查看和管理系统通知。

dove notify [action] [id] [options]

Actions

Action Description
list 列出通知（默认 action）
read <id> 标记通知为已读
config 配置通知偏好
history 查看通知历史

选项

Flag Description Default
--json JSON 格式输出 false
-n, --limit <n> 列出条数 20

示例

# 列出未读通知
dove notify

# 列出最近 50 条
dove ntf list -n 50

# 标记为已读
dove ntf read ntf_abc123

# 配置通知偏好
dove ntf config

# 查看通知历史
dove ntf history --json

dove team

团队模型配置命令，管理团队共享的模型和提供商设置。

dove team [action] [name] [options]

Actions

Action Description
show / ls / list 查看团队配置
default <name> 设置默认团队模型
add <name> 添加团队模型配置
remove / rm <name> 移除团队模型
update / edit <name> 更新团队模型配置
reset 重置为默认配置

选项

Flag Description Default
-m, --model <model> 模型名称 -
-p, --provider <name> 提供商 -

示例

# 查看团队配置
dove team list

# 添加团队模型
dove team add "主力模型" -p anthropic -m claude-opus-4-20250514

# 设为默认
dove team default "主力模型"

# 更新配置
dove team edit "主力模型" -m claude-sonnet-4-20250514

# 移除模型
dove team rm "旧模型"

# 重置为默认
dove team reset

dove app

扩展应用管理命令，安装、开发、提交和审核扩展应用。

dove app [action] [options]

Actions

Action Description
install <name> 安装扩展
uninstall <name> 卸载扩展
info <name> 查看扩展信息
list 列出已安装扩展
search <query> 搜索扩展市场
create <name> 创建扩展项目脚手架
validate 验证扩展配置
dev 开发模式运行
submit 提交扩展审核
review <id> 审核扩展（管理员）
pending 查看待审核列表（管理员）
check <id> 检查扩展状态
publish <id> 发布扩展（管理员）
download <name> 下载扩展源码

示例

用户操作

# 搜索扩展
dove app search "视频处理"

# 安装扩展
dove app install video-processor

# 查看已安装
dove app list

# 查看扩展信息
dove app info video-processor

# 卸载扩展
dove app uninstall video-processor

开发者工作流

# 1. 创建扩展脚手架
dove app create my-extension

# 2. 进入项目目录开发
cd my-extension

# 3. 验证 manifest 和权限配置
dove app validate

# 4. 开发模式运行（热重载）
dove app dev

# 5. 验证通过后提交审核
dove app submit

# 6. 查看审核状态
dove app check ext_abc123

管理员操作

# 查看待审核列表
dove app pending

# 审核扩展
dove app review ext_abc123

# 发布到市场
dove app publish ext_abc123

提示：dove app create、dove app validate 和 dove app dev 是免认证命令，本地开发无需登录。

安全提示：安装第三方扩展前请仔细阅读其权限声明。扩展会获得声明范围内的系统访问权限。
